Git ist ein beliebtes VCS und wurde 2005 von Linus Torvalds, dem Entwickler hinter Linux, erstellt.
Es konnte sich aufgrund seiner hohen Sicherheit und Effizienz durchsetzen und Verteilte, gleichzeitig ablaufende Arbeitsvorgänge. Die Versionskontrolle geschieht hier durch das Branch System.
Will ein Entwickler z.B. änderungen an einem Code machen kann dieser eine Kopie von der neusten Version machen und dann dieses in einen unabhängigen Branch bearbeiten und testen.
Diese änderungen werden zuallererst im Lokalen Repository gespeichert und werden dann dem Team per pull request zur Verfügung gestellt.
Git kann und wird dabei häufig zusammen mit GitHub verwendet. Es dient hierbei als remote repository und erleichtert entwicklern die Zusammenarbeit.

Befehl Funktion
Git add Fügt änderung zum Stage hinzu, werden im nächsten Commit aufgenommen
Git commit Fügt Änderung ins lokale Repository hinzu
Git push Pusht branch ins Remote repository, kann jetzt von anderen Entwicklern im Team eingesehen werden
Git merge Integriert Branch in master branch
